## Nom `name`

<ParamField path="name" type="string" required>
  Le nom utilisé par le personnage dans le chat, ainsi que le nom que les autres utilisateurs verront lorsque vous le publierez sur le marché de découverte.
</ParamField>

Le nom peut être un prénom ("Lilia"), un nom complet ("Albert Einstein") ou un surnom. Il ne doit pas dépasser 64 caractères, car plus un nom est long, plus il est difficile à retenir.

<Frame>
  <img src="https://oss.vidol.chat/assets/613445d55d376eb45fb94ae13f3834ff.webp" />
</Frame>

### Définition par le nom

Le nom lui-même peut avoir un impact énorme sur le personnage, surtout lorsque celui-ci manque d'autres détails ou que le nom est bien connu.

Par exemple, si votre personnage a un nom largement reconnu, un nom qui apparaît plusieurs fois au cours de l'entraînement du système, ce nom peut définir son comportement de base. Le prénom et le nom de famille aident à l'identification, donc "Albert Einstein" sera plus efficace que simplement "Albert", ce dernier étant flou sans plus de détails.

### Unicité

Le nom n'a pas besoin d'être unique, donc plusieurs personnages peuvent avoir le même nom. Nous utilisons l'AgentID pour garantir l'unicité globale.

## Avatar `avatar`

<ParamField path="avatar" type="file" required>
  L'image de l'avatar sera utilisée avec votre nom et d'autres éléments pour rendre votre personnage visuellement attrayant et unique.
</ParamField>

<Info>
L'avatar peut être dans plusieurs formats (png, gif, jpeg), et idéalement, il devrait être une image carrée de 512x512 pixels.
</Info>

L'avatar que vous téléchargez sera recadré en cercle dans le contexte du chat :

</Frame>

Sur l'écran principal sous forme de carte d'information du personnage, il sera affiché sous forme d'image carrée non recadrée :

## Description `description`

<ParamField path="description" type="string">
  La description apparaît sous le nom du personnage et peut ajouter plus d'informations à votre personnage (la partie sous le nom).
</ParamField>

<Frame>
  <img src="https://oss.vidol.chat/assets/613445d55d376eb45fb94ae13f3834ff.webp" />
</Frame>

La description fournit un moyen concis de décrire le personnage et le contexte, aidant les gens à comprendre le personnage, surtout lorsque le nom du personnage est lui-même ambigu.

## Salutation `greeting`

<ParamField path="greeting" type="string">
  La première phrase que le personnage dira en commençant une nouvelle conversation, la salutation ne doit pas dépasser 200 caractères.
</ParamField>

<Frame>
  <img src="https://oss.vidol.chat/assets/3c257ce957b6775b41d259feda3bb708.webp" />
</Frame>

La salutation peut donner aux utilisateurs une idée de ce que c'est que d'interagir avec le personnage, et peut même introduire un jeu ou suggérer la prochaine action. Par exemple, dans un jeu d'aventure textuel, la salutation est utilisée non seulement pour définir le personnage, mais aussi pour introduire aux utilisateurs ce dont ils vont discuter.

<ParamField path="readme" type="string">
  L'explication du personnage est utilisée pour compléter et informer les utilisateurs sur ce personnage, les aidant à comprendre les instructions d'utilisation du personnage, comme les précautions d'utilisation du modèle, etc.
</ParamField>
