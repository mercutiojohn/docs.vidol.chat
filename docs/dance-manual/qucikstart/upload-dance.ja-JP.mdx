必要なファイルを準備したら、ウェブサイトが提供するアップロード機能を使って、関連するダンスファイルをバックエンドにアップロードし、審査を受けることができます。

## アップロード手順

<Steps>
  <Step title="ダンスインターフェースを開く">
   Vidolインターフェースを開いたら、右側の `展開` ボタンをクリックし、 `ダンス` タブに切り替え、次に `ダンスマーケット` ボタンをクリックします。以下の図を参照してください：

![](https://oss.vidol.chat/dance-manual-step-1.png)

  </Step>
  <Step title="ダンスを作成するをクリック">
   ダンスマーケットを開いたら、 `ダンスを作成する` ボタンをクリックすると、ダンス作成フォームが開きます：

```
![](https://oss.vidol.chat/dance-manual-step-2.png)
```

  </Step>
  <Step title="ダンスフォームに記入する">
    次に、前のステップで準備したダンスファイルをアップロードし、ダンスID、名前、および関連する説明を追加します。詳細なダンスフィールドの説明は[ダンスパラメータの説明](/dance-manual/qucikstart/dance-fields)を参照してください。記入が完了したら、下の `アップロードして提出` ボタンをクリックします：
   ![](https://oss.vidol.chat/dance-manual-step-3.png)
  </Step>
</Steps>

## パラメータの説明

<ParamField path="danceId" type="string" required>
  ダンスの一意性を識別するためのIDです。
</ParamField>

<ParamField path="name" type="string" required>
  ダンスの名前で、64文字を超えないでください。
</ParamField>

<ParamField path="src" type="string" required>
  アニメーションファイルのURLです。
</ParamField>

<ParamField path="audio" type="string" required>
  音声ファイルのURLで、現在は `.mp3` と `.wav` フォーマットがサポートされています。
</ParamField>

<ParamField path="cover" type="string" required>
  ダンスのカバー画像で、推奨サイズは `512*512` です。
</ParamField>

<ParamField path="thumb" type="string" >
  ダンスのプレビューで、ダンスのカバーから自動生成されます。
</ParamField>

<ParamField path="readme" type="string" >
  説明文、例えばアニメーションリソースのURLや使用上の注意点などです。
</ParamField>

<ParamField path="author" type="string">
  制作者の名前で、このフィールドは審査が完了した後に自動的にGitHubアカウント名を取得します。
</ParamField>

<ParamField path="homepage" type="string">
  制作者のホームページで、このフィールドは審査が完了した後に自動的にGitHubアカウントのURLを取得します。
</ParamField>

<ParamField path="createAt" type="string">
  作成日で、このフィールドは審査が完了した後に自動的に現在の日付が生成されます。
</ParamField>

<RequestExample>
